# 前端用户流程

## 市民

进入智能对话 → Rasa 引导补齐诉求 → Action Server 创建 PostgreSQL 工单 → 消息突出显示完整工单编号 → 直接进入详情或“我的工单” → 查看对外可见的流转时间线 → 对处理结果评价。满意/基本满意直接办结；不满意必须填写原因并退回部门重办。

## 坐席

登录 → 查看待受理/可见工单 → 受理或选择标准原因不予受理 → 选择启用部门并派发 → 必要时修改联系方式。所有操作提交备注与当前版本。

## 部门人员

登录 → 查看本部门工单或切换“分派给我的工单” → 打开详情 → 开始处理并认领承办人 → 添加一条或多条内部处理记录 → 填写结果摘要、处理措施、解决情况、公开答复和可选内部备注 → 提交待市民确认 → 查看历史记录。后端限制只能处理本部门工单，个人筛选由服务端执行。

## 管理员

登录 → 查看真实聚合看板 → 查询全部工单并执行合法流转 → 必要时填写公开依据代市民办结 → 管理用户和部门启停 → 查看审计日志。

## 异常路径

- 401：清理 token 并回到登录页。
- 403：显示无权访问；不尝试通过前端绕过。
- 404：显示资源不存在。
- 409：提示数据被更新并刷新详情。
- 422：展示后端业务/字段校验消息。
- Backend/Rasa 不可用：安全错误页或聊天重试，不显示技术堆栈。
