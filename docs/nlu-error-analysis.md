# 第三轮中文 NLU 错误分析

## 评估设计

- 冻结盲测：`tests/nlu_blind_round3.yml`，共 120 条，投诉/建议/咨询/求助各 30 条。
- 数据均为脱敏或虚构表达，包含长句、口语、省略、多事件，以及道路、环境、物业、交通、教育、医疗、社区服务和政务办理。
- 61 个地点标注覆盖道路、小区、村镇、学校、医院、政务设施和相对地点。
- 优化前模型：`models/tingting-round2-final.tar.gz`。
- 优化后模型：`models/tingting-round3.tar.gz`。
- 防泄漏脚本输出：`blind=120 training=708 exact_overlap=0`。盲测原句没有并入训练集。

复现命令见 README。原始 JSON、混淆图和错误文件位于 `results/round3-nlu-before/` 与 `results/round3-nlu-after/`。

## 最终错分样本

| 真实 | 预测 | 样本 | 原因 |
|---|---|---|---|
| 投诉 | 咨询 | 昨天办医保转移，窗口收了材料后又说全部要重交 | “医保转移/材料”具有强办事咨询特征；句子未出现评价词，模型未充分利用“收了后又要求重交”的已发生不当行为。 |
| 建议 | 咨询 | 能不能把各项补贴的申请条件做成一张对比表 | 疑问句式与“申请条件”偏向咨询，建议的交付物意图被掩盖。 |
| 建议 | 投诉 | 翠竹广场晚上人多，希望多放几个分类垃圾桶 | 当前问题背景与改善方案混合，模型更关注“人多/垃圾”问题场景。 |

优化前另有“证件能否提前换发”“周日能否领证”被分到求助/建议，以及三条制度改进建议被分到投诉；通用对比数据已修复这些样本，未复制盲测原句。

## 地点实体结果

| extractor / 指标 | Precision | Recall | F1 |
|---|---:|---:|---:|
| 优化前 DIET | 62.96% | 27.87% | 38.64% |
| 优化后 DIET | 61.54% | 26.23% | 36.78% |
| 优化后 Regex/Lookup | 61.29% | 93.44% | 74.03% |

Regex/Lookup 显著补足召回，但 DIET 自身没有改善，因此报告不把 Regex 的提升描述为统计模型提升。第一版贪婪后缀正则曾得到 Precision 41.86%、Recall 59.02%、F1 48.98%，错误分析发现会把“市场污水直接排到路”等长片段当地点，随后改为：

- 道路、小区、社区、村镇、学校、医院、政务设施只在句首或“在/到/去/从”等地点上下文匹配；
- 相对地点只匹配“我家门口、小区东门、学校旁边”等明确结构；
- 行政区划和常见道路/社区/学校/医院使用独立 Lookup；
- Action 对多个 extractor 的候选进行保守归一化，去掉“建议在/希望”等话语前缀，拒绝单独“医院/学校/社区/农村”等泛称；不可靠时不填槽并继续表单追问。

### 剩余地点错误

- 泛称误报：例如“希望医院提供……”仍可能被 Regex 报告计为地点；Action 入槽前会拒绝清洗后仅为“医院”的候选。
- 标注边界：盲测采用保守标注，部分泛指“学校门口/社区服务中心”未标地点，而规则按相对地点需求会提取，因此报告精确率受到影响。本轮没有在看到结果后修改盲测标签。
- 多 extractor 重叠：DIET 与 Regex 可能同时返回道路的不同跨度；Action 优先 Regex 并选择归一化后的完整候选。
- 未覆盖的真实地址、门牌号和歧义名词不会被推断为具体地址，仍由表单追问。

## 错误原因分类与后续数据

| 类别 | 表现 | 需要补充的数据 |
|---|---|---|
| 已发生办事不当 vs 政策咨询 | 行政术语压过投诉事件结构 | “已收件后反复补件/口径变化”与正常材料咨询成对样本 |
| 建议问句 vs 咨询问句 | “能不能 + 条件”被识别为咨询 | 含可交付改进物的疑问式建议；只询问事实/资格的咨询对照 |
| 问题背景 + 改善方案 | 当前负面背景压过建议目的 | 长句中前半问题、后半提出公共改进方案的样本 |
| 机构泛称 vs 具体地点 | “医院/学校”可指类别而非地点 | 有名称、无名称、相对地点和职能对象的对照实体标注 |
| 多地点/交叉口边界 | DIET 分段、Regex 完整跨度不一致 | 道路交叉口、学校旁道路、医院多个院区的统一边界规范 |

建议下一轮先扩充成对最小差异数据和双人复核实体标注，再考虑调参；当前没有引入 BERT、LLM 或其他主模型升级。
