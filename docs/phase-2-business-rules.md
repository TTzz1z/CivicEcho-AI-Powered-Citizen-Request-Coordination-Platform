# 第二阶段业务规则与状态流转

## 1. 诉求分类

- 分类最多三级，编码全局唯一，同一上级下名称唯一。
- 工单选择没有启用子分类的末级分类；停用分类不能用于新受理，但历史工单继续展示原分类。
- 上级停用前必须先停用所有启用子分类；存在子分类时不得造成超过三级的移动。
- 末级分类可配置默认责任部门、默认受理时限和默认办理时限。默认部门用于派发预选，坐席仍需确认派发。
- 只有管理员可新增、编辑、启用和停用分类；所有变更写入 `audit_logs`。

## 2. 优先级

正式优先级为 `normal`（普通）、`expedited`（加急）、`urgent`（紧急）、`major`（重大事件）。

- 市民或智能对话提交的紧急程度只保存为 `requested_priority`，不能直接改变正式优先级。
- 新工单正式优先级固定为普通；坐席或管理员在受理动作中确认末级分类和正式优先级。
- 受理动作记录确认人和确认时间，并写入审计。
- SLA 系数：普通 100%、加急 75%、紧急 50%、重大事件 25%，结果向最近一分钟取整且至少一分钟。
- 旧值兼容：`low/normal → normal`、`high → expedited`、`urgent → urgent`；原值保存在 `requested_priority`。

## 3. SLA 计时

- 新建时使用兼容默认值：2 小时应受理、3 天应办结。
- 坐席受理后，按确认分类的默认时限和正式优先级重新计算：`应受理时间 = 创建时间 + 分类受理时限 × 系数`，`应办结时间 = 实际受理时间 + 分类办理时限 × 系数`。
- 待受理阶段以应受理时间判断；受理后至部门提交处理结果前以应办结时间判断。
- `due_soon` 定义为剩余不超过 4 小时；`overdue` 为当前活动截止时间已过；暂停时显示 `paused`。
- 部门人员仅能暂停本部门处于已受理、已派发或处理中的工单，管理员可处理全局工单。暂停必须填写原因。
- 恢复时按实际暂停秒数顺延两个截止时间，累计暂停秒数保留。暂停、恢复均使用版本号控制并写入历史和审计。
- 市民可催办本人未终态工单；坐席、责任部门人员和管理员可在权限范围内催办。每次催办增加计数、写公开办理记录并审计。
- 已解决、已办结、不予受理不能催办；已解决、已办结和不予受理不能暂停。

## 4. 状态与角色

| 当前状态 | 动作 | 目标状态 | 角色 | 第二阶段附加规则 |
|---|---|---|---|---|
| 待受理 | 受理 | 已受理 | 坐席、管理员 | 确认末级分类和正式优先级，生成 SLA |
| 待受理 | 不予受理 | 不予受理 | 坐席、管理员 | 仍按一期标准原因规则 |
| 已受理 | 派发 | 已派发 | 坐席、管理员 | 分类默认部门作为预选值 |
| 已派发 | 开始处理 | 处理中 | 责任部门、管理员 | 部门人员自动认领 |
| 已受理/已派发/处理中 | 暂停/恢复 | 状态不变 | 责任部门、管理员 | 原因必填，恢复顺延截止时间 |
| 未终态 | 催办 | 状态不变 | 权限范围内四角色 | 计数、公开历史、审计 |
| 处理中 | 提交结果 | 待市民确认 | 责任部门、管理员 | 停止办理 SLA 比较 |
| 待市民确认 | 满意反馈 | 已办结 | 工单所属市民 | 记录实际办结时间 |
| 待市民确认 | 不满意反馈 | 处理中 | 工单所属市民 | 重新进入办理 SLA，不重置原截止时间 |

## 5. 数据兼容

Alembic `0005` 只新增表、可空字段和带默认值字段。旧工单分类允许为空，并回填兼容 SLA；旧优先级按上述映射转换，不删除历史、办理记录或审计。降级迁移会把 `expedited` 映射回 `high`、`major` 映射回 `urgent`。
